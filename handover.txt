Auto-reply (engine moves once)

Replaced older rAF/effect race with maybeAutoReply(), which:

Fires only after a user move

Ensures it’s engine’s turn

Ensures only once per ply with lastAutoReplyAtPly.current

This eliminated the “engine moves twice” bug.

Accuracy & Elo

Accuracy (per side): Lichess-style ΔWin% loss vs best (book moves ignored)

Win probability from WHITE-POV centipawns: wp = 1 / (1 + e^(-(cp/130)))

Mover’s loss = max(0, wpBestMover - wpAfterMover)

Accuracy% = 100 * (1 - averageLoss)

Elo estimate (single number) from accuracy (Chess.com-style):

Elo = round(32.82 * ln(acc/(100-acc)) + 1850)   // acc in 1..99


Avg CPL is also computed and shown (excluding Book).

Tagging

Book: Tag all moves before deviation as 'Book'.

Non-book moves use ΔWin% loss thresholds for human-like tagging:

<0.5% → Best

<2% → Good

<10% → Mistake

≥10% → Blunder

Navigation

Keyboard: ← / → / Home / End navigates via a single rebuildTo(n) (authoritative), which rebuilds a fresh Chess() position to n plies and updates FEN → board always follows the row selection.

Move list: clicking a row calls onRebuildTo(rowIndex+1).

Key files & responsibilities
webui/src/App.tsx (authoritative state/logic)

Keeps gameRef (Chess), fen, movesUci, ply

Orchestrates analysis:

For each ply, calls backend to analyze before and after

Computes best, cpBefore/cpAfter (WHITE POV), cpl (mover POV), tag

Computes accuracy, avg CPL, Elo

Auto-reply guard (maybeAutoReply) prevents double engine moves

Debounced position eval (lightweight, separate from analysis) for eval bar

Keyboard navigation → always through rebuildTo() (which blocks eval temporarily)

Important sign conventions inside analyzePgn():

bestCp: eval before the move = mover POV (as returned by engine at side-to-move)

afterCpRaw: eval after the move = opponent POV → flip (-afterCpRaw) to get mover POV

WHITE POV copies for UI:

cpBeforeWhite = mover==='w' ? bestCp : -bestCp

cpAfterWhite = mover==='w' ? -afterCpRaw : afterCpRaw (because afterCpRaw is opp POV)

CPL (mover POV): cpl = max(0, bestForMover - afterForMover)

webui/src/openings/matcher.ts

initOpenings('/eco.json') – must be reachable from Vite under /eco.json (file is in webui/public/eco.json)

Common pitfalls & fixes

Openings don’t load: Ensure webui/public/eco.json exists and is reachable at /eco.json. Check DevTools → Network.

All moves tagged "Mistake": This was due to old CPL-only thresholds. We now use ΔWin% thresholds. Verify analyzePgn() uses tagFromLoss(loss) and not the old function.

Book tags missing: Ensure await initOpenings('/eco.json') is called before analysis; bookUntil = deviationIndex(movesUci) with a fallback to openingAt if needed.

Auto-reply does two moves: Fixed by using maybeAutoReply() and lastAutoReplyAtPly. Ensure no leftover effect/rAF path triggers engine move elsewhere.

Board not following on ←/→: Only change ply via rebuildTo(n). We attach global key handlers (capture) and call rebuildTo() so gameRef + fen stay in sync.

Important formulas & thresholds (so successors don’t have to hunt)
Win probability (WHITE POV) from centipawns
wp = 1 / (1 + exp(-(clamp(cp, -1000, 1000) / 130)))

ΔWin% loss for mover (used for both accuracy and tags)

Convert cpBeforeWhite, cpAfterWhite to win prob for mover (flip if mover is Black).

loss = max(0, wpBestMover - wpAfterMover)

Accuracy (%), per side
accuracy = 100 * (1 - average(loss for side))

Elo estimate from accuracy (Chess.com-style)
Elo = round(32.82 * ln(acc/(100-acc)) + 1850), acc in [1..99]

Tag thresholds (ΔWin% loss)

<0.5% → Best

<2% → Good

<10% → Mistake

≥10% → Blunder

(Book moves tagged 'Book' up to deviation index)

Pseudocode checkpoints (so they can verify quickly)

Auto-reply guard

async function maybeAutoReply() {
  if (!autoReply || analyzing || suspendEval || engineBusy) return;
  const engineSide = userSide === 'w' ? 'b' : 'w';
  if (gameRef.current.turn() !== engineSide) return;
  if (lastAutoReplyAtPly.current === ply) return;
  lastAutoReplyAtPly.current = ply;
  setSuspendEval(true);
  try { await engineMove(); } finally { setSuspendEval(false); }
}


Rebuild navigation

function rebuildTo(n:number) {
  setSuspendEval(true);
  lastAutoReplyAtPly.current = -1;
  const g = new Chess();
  for (let i = 0; i < Math.min(n, movesUci.length); i++) safeMove(g, movesUci[i]);
  gameRef.current = g;
  setPly(n);
  setFen(g.fen());
  setTimeout(() => setSuspendEval(false), 0);
}


Analysis (sign hygiene)

// before
bestCp // mover POV (side-to-move eval)
best = before.bestMove

// after
afterCpRaw // opponent POV
afterForMover = -afterCpRaw

// white POV copies for UI
cpBeforeWhite = mover==='w' ? bestCp      : -bestCp
cpAfterWhite  = mover==='w' ? -afterCpRaw :  afterCpRaw

// CPL (mover POV)
cpl = max(0, (bestCp) - (afterForMover))

Contact points / expectations for successor

If anything drifts (e.g., “all moves tagged Mistake” or “Book never appears”), the first place to check is analyzePgn() sign usage and that initOpenings('/eco.json') is invoked before analysis.

Performance-sensitive areas: avoid spamming eval calls; we use a 120ms debounce and suspendEval while engine is moving or we’re rebuilding positions.

Keep backend simple: single route /api/engine/analyze, movetimeMs tunable; multiPV typically 1–3.
